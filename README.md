**DirectX12PioneerEngine** 是一个用于学习现代渲染技术的个人项目，旨在通过实践深入理解 DirectX 12 和实时渲染管线。作为职业转型（从 WPF 开发转向 3D 图形开发）的作品展示，该项目验证了我对图形学理论的理解，并为后续开发更完整的游戏引擎打下基础。

> “Pioneer” 意为先行者——本项目是技术探索的先驱，未来将基于此经验开发一个真正的游戏引擎。

---

## 📖 项目背景

作为一名有三年 WPF 开发经验的软件工程师，我在工作中频繁接触 3D 可视化业务，逐渐对实时渲染产生浓厚兴趣。本引擎是我学习 DirectX 12 的实践平台：以《DirectX12 3D游戏开发实战》第九章的 Crate 示例为起点，逐步扩展出完整的渲染系统和工具链，最终将零散知识整合为一个可工作的引擎原型。

**项目起始时间**：2025年9月

---

## 🏗️ 技术架构

项目采用**前后端分离**的架构，充分发挥不同语言的优势：

- **底层渲染核心（C++ / DirectX 12）**  
  由多个静态库组成，共同实现渲染、资源、场景等核心系统。依赖以下第三方库，所有库均放置在项目根目录的 `ThirdParty` 文件夹中：
  - **Boost**：提供智能指针、文件系统等基础支持。
  - **yaml-cpp**：用于场景和配置文件的序列化。
  - **Assimp**：模型加载与转换。

- **上层交互界面（C# / WPF）**  
  遵循 **MVVM 模式** 构建编辑器界面，利用 WPF 强大的数据绑定和控件库，快速构建友好的用户界面。

- **互操作层**  
  通过 P/Invoke 和 C++/CLI 实现 C++ 核心与 C# 前端的无缝通信，并利用 WinForms 控件（`ViewportLib`）将 DirectX 12 渲染视图嵌入 WPF。

## 📦 项目结构

项目基于 Visual Studio 2022 构建，包含以下核心库/可执行项目：

| 库/项目名 | 职责 |
|-----------|------|
| **Crate** | DirectX 底层 API 封装，源自《DirectX12 3D游戏开发实战》第九章示例，作为图形抽象层。 |
| **Rendering** | 基于 Crate 提供高级渲染功能：材质、光照、网格、渲染流水线自定义、Shader 文件加载等。 |
| **Engine** | 核心引擎层，包含**实体-组件系统（ECS）**、渲染系统、资源系统、创建系统，并暴露 C# 可调用的接口。 |
| **Editor** | 项目是一个基于 **MVVM 模式** 的 WPF 应用程序，提供完整的编辑器界面。通过互操作层（P/Invoke、C++/CLI 和 `ViewportLib`）与 C++ 核心通信，实现场景编辑、实时预览等功能。|
| **Common** | 通用帮助函数，被所有库共享。 |
| **Math** | 数学库，提供向量、矩阵、变换等常用函数。 |
| **EngineException** | 自定义异常类型，便于错误处理。 |
| **ViewportLib** | WinForms 库，用于在 WPF 中嵌入 DirectX 12 渲染视图（利用窗口句柄）。 |
| **DirectX12PioneerEngine.UI** | WPF 交互层，遵循 MVVM 模式，提供 Editor 控件的样式和主题，构建用户界面。 |

## ✨ 已完成功能

### 渲染特性
- **延迟渲染**（Deferred Rendering）管线，支持多光源。
- **基于物理的渲染**（PBR）：实现金属/粗糙度工作流，支持 HDR 环境光照。
- **平行光照**（Directional Light）与基本光照模型。
- **阴影映射**（Shadow Mapping），实现动态阴影。
- **屏幕空间环境光遮蔽**（SSAO），增强场景深度感。
- **快速近似抗锯齿**（FXAA），平滑边缘锯齿。

### 引擎核心
- **实体-组件系统（ECS）**：灵活组织游戏对象，支持动态增删组件。
- **资源管理**：模型、纹理、Shader 的异步加载与生命周期管理。
- **场景图**：支持层级变换与场景序列化（YAML 格式）。

### 编辑器
- **视口预览**：实时渲染窗口，支持摄像机操控。
- **属性面板**：编辑实体变换、组件参数（数据绑定基于 MVVM）。
- **资源浏览器**：预览和导入模型、纹理。
- **日志窗口**：显示引擎运行状态与错误信息。

## 🔧 构建与运行

### 环境要求
- Windows 10/11（需支持 DirectX 12）
- Visual Studio 2022（或 2019）  
  - 使用“使用 C++ 的桌面开发”工作负载  
  - 安装“.NET 桌面开发”工作负载（用于 WPF）
- Windows SDK（10.0.20348 或更高）
- DirectX 12 运行时（系统自带）

### 第三方库依赖
项目依赖以下 C++ 库，需放置在解决方案根目录的 `ThirdParty` 文件夹中。推荐目录结构如下：
- DirectX12PioneerEngine/
- ├── ThirdParty/
- │ ├── boost/ # Boost 头文件和库（或源码）
- │ ├── yaml-cpp/ # yaml-cpp 头文件和库（或源码）
- │ └── assimp/ # Assimp 头文件和库（或源码）
- ├── Crate/
- ├── Rendering/
- └── ... 其他项目文件夹
